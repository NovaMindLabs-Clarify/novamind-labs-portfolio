# PRD — Лендинг «Стоматологическая клиника»

## 1. Обзор и цель проекта
Портфолио-лендинг для стоматологической клиники: демонстрация умения делать быстрый, конверсионный one-page сайт для медицинского локального бизнеса. Показывается потенциальным заказчикам как образец.

**Основная цель конверсии:** клик по CTA «Записаться на приём» / звонок по телефону.
**Демонстрируемые компетенции:** современный дизайн 2026, адаптивность, двуязычность EN/RU, живые интерактивные элементы без бэкенда, работа с «доверительным» медицинским контентом.

## 2. Границы проекта
**В скоуп:** одностраничный сайт (one-pager) с якорной навигацией, статичный контент, переключатель EN/RU на клиенте, форма онлайн-записи (клиентская валидация, без реальной отправки — success-состояние в UI), базовое SEO, адаптив 320–1920px.
**Вне скоупа:** реальная интеграция с медицинской CRM/расписанием врачей и оплатой, CMS/админка, мультистраничность, реальные пользовательские аккаунты, обработка персональных медицинских данных.

## 3. Целевая аудитория
Взрослые пациенты, ищущие стоматолога рядом с домом/работой; ключевой барьер — страх перед лечением, важны доверие и прозрачность цен. См. `PLAN.md` — полный портрет и палитра/шрифты.

## 4. Технический стек
- **HTML5 + CSS3 (без препроцессоров) + Vanilla JS (ES6+)**, без сборки — открывается напрямую в браузере, легко деплоится на любой статичный хостинг.
- Шрифты — Google Fonts (Inter/Manrope), подключение через `<link>`.
- Иконки — инлайн SVG (без иконочных шрифтов).
- i18n — словари `js/i18n/en.js`, `js/i18n/ru.js` + атрибуты `data-i18n`, язык по умолчанию **EN**, выбор сохраняется в `localStorage`.
- Адаптив — mobile-first, CSS Grid/Flexbox, брейкпоинты 375 / 768 / 1024 / 1440.
- Структура файлов:
  ```
  5_site/
    index.html
    css/style.css
    js/main.js
    js/i18n.js
    assets/img/
  ```

## 5. Функциональные требования по секциям
1. **Header** — лого, якорные ссылки (Услуги/Врачи/Цены/Отзывы/Контакты), переключатель EN/RU, плашка «Emergency» с номером телефона, CTA «Book Appointment», sticky, бургер-меню на мобильном.
2. **Hero** — заголовок ≤10 слов про безболезненное/заботливое лечение, подзаголовок, кнопка «Book Online» (основная) + «Call Now» (вторичная), фото довольного пациента/клиники.
3. **Trust bar** — сертификаты/лицензии, лет опыта, довольных пациентов, рейтинг Google — горизонтальная строка/иконки.
4. **Услуги** — карточки направлений (терапия/имплантация/ортодонтия/эстетика/детская стоматология) с кратким описанием.
5. **Врачи** — карточки специалистов (фото, имя, квалификация, стаж, кнопка «Book with Dr. [имя]»), горизонтальный скролл на мобильном.
6. **Галерея до/после** — сетка 6–8 фото примеров работ.
7. **Отзывы** — 3–4 карточки (фото/аватар, имя, текст, звёзды, в идеале — стилизация под Google-отзывы).
8. **Цены/прозрачность** — таблица/карточки с базовыми ценами на популярные услуги, упоминание рассрочки/страховки.
9. **FAQ** — аккордеон, 4–6 вопросов (больно ли, сколько стоит первый приём, экстренные случаи).
10. **Запись/контакты** — форма онлайн-записи (имя, телефон, услуга, желаемая дата, кнопка с валидацией и success-сообщением), карта (iframe Google Maps placeholder), адрес, часы работы, телефон.
11. **Footer** — номера лицензий, соцсети, копирайт, повтор навигации.

## 6. Нефункциональные требования
- **Производительность:** изображения в WebP, `loading="lazy"` для контента ниже первого экрана, минимум внешних зависимостей, цель — загрузка <2 сек на мобильном.
- **SEO:** уникальные `<title>`/`<meta description>`/Open Graph теги для EN и RU версий контента, семантическая разметка, alt-тексты на всех изображениях, разметка структуры (local business/medical) как задел под schema.org.
- **Доступность (a11y):** контраст текста ≥ WCAG AA, вся навигация и аккордеон доступны с клавиатуры, `aria-label` на иконках-кнопках, крупные тач-таргеты для CTA (важно для тревожной аудитории).
- **Браузеры:** актуальные Chrome/Firefox/Safari/Edge, мобильный Safari/Chrome (iOS/Android).
- **Аналитика:** место для сниппета Google Analytics/Яндекс.Метрика (закомментировано, не активно по умолчанию).

## 7. Контент
Копирайт готовится в двух версиях (EN — основной, RU — перевод/адаптация), тон — спокойный, заботливый, снимающий тревожность. Для портфолио допустимы качественные стоковые/AI-фото команды и клиники, подобранные под палитру и настроение доверия.

## 8. Критерии приёмки (Definition of Done)
- [ ] Все 11 секций реализованы и наполнены контентом на EN и RU
- [ ] Переключатель языка работает без перезагрузки страницы, дефолт — EN
- [ ] Адаптивность 320–1920px без горизонтального скролла

- [ ] Lighthouse (мобильный): Performance/Accessibility/SEO/Best Practices ≥ 90
- [ ] Нет ошибок в консоли браузера
