# PRD — Лендинг «Салон красоты»

## 1. Обзор и цель проекта
Портфолио-лендинг для салона красоты: демонстрация умения делать быстрый, конверсионный one-page сайт для локального бизнеса. Показывается потенциальным заказчикам как образец.

**Основная цель конверсии:** клик по CTA «Записаться онлайн».
**Демонстрируемые компетенции:** современный дизайн 2026, адаптивность, двуязычность EN/RU, живые интерактивные элементы без бэкенда.

## 2. Границы проекта
**В скоуп:** одностраничный сайт (one-pager) с якорной навигацией, статичный контент, переключатель EN/RU на клиенте, форма/виджет записи (клиентская валидация, без реальной отправки — success-состояние в UI), базовое SEO, адаптив 320–1920px.
**Вне скоупа:** реальная интеграция с системой онлайн-записи и оплатой, CMS/админка, мультистраничность, реальные пользовательские аккаунты.

## 3. Целевая аудитория
Женщины 20–50 лет, ищущие салон для регулярного ухода (волосы, ногти, косметология), ценящие эстетику и удобную запись. См. `PLAN.md` — полный портрет и палитра/шрифты.

## 4. Технический стек
- **HTML5 + CSS3 (без препроцессоров) + Vanilla JS (ES6+)**, без сборки — открывается напрямую в браузере, легко деплоится на любой статичный хостинг.
- Шрифты — Google Fonts (Playfair Display + Inter/Poppins), подключение через `<link>`.
- Иконки — инлайн SVG (без иконочных шрифтов).
- i18n — словари `js/i18n/en.js`, `js/i18n/ru.js` + атрибуты `data-i18n`, язык по умолчанию **EN**, выбор сохраняется в `localStorage`.
- Адаптив — mobile-first, CSS Grid/Flexbox, брейкпоинты 375 / 768 / 1024 / 1440.
- Структура файлов:
  ```
  3_site/
    index.html
    css/style.css
    js/main.js
    js/i18n.js
    assets/img/
  ```

## 5. Функциональные требования по секциям
1. **Header** — лого, якорные ссылки (Услуги/Мастера/Галерея/Отзывы/Контакты), переключатель EN/RU, CTA «Book Now», sticky, бургер-меню на мобильном.
2. **Hero** — заголовок ≤10 слов про красоту/уверенность, подзаголовок, кнопка «Book Online» (основная) + «View Services» (вторичная), фоновое фото интерьера/процедуры с оверлеем.
3. **Trust bar** — 3–4 цифры (лет на рынке, довольных клиенток, рейтинг), горизонтальная строка/иконки.
4. **О нас** — блок текст+фото (2 колонки на десктопе, стек на мобильном), философия салона и используемые бренды косметики.
5. **Услуги и цены** — карточки по направлениям (волосы/ногти/косметология/спа) с ценами.
6. **Мастера** — карточки специалистов (фото, имя, специализация, кнопка «Book with [имя]»), горизонтальный скролл на мобильном.
7. **Галерея до/после** — сетка 6–8 фото (CSS grid).
8. **Отзывы** — 3–4 карточки (фото/аватар, имя, текст, звёзды), горизонтальный скролл на мобильном.
9. **FAQ** — аккордеон, 4–6 вопросов (часы работы, отмена записи, абонементы/пакеты услуг).
10. **Запись/контакты** — блок с виджетом-заглушкой записи (UI-имитация выбора даты/времени/мастера), карта (iframe Google Maps placeholder), адрес, часы работы, телефон.
11. **Footer** — соцсети (Instagram и др.), копирайт, повтор навигации.

## 6. Нефункциональные требования
- **Производительность:** изображения в WebP, `loading="lazy"` для контента ниже первого экрана, минимум внешних зависимостей.
- **SEO:** уникальные `<title>`/`<meta description>`/Open Graph теги для EN и RU версий контента, семантическая разметка, alt-тексты на всех изображениях.
- **Доступность (a11y):** контраст текста ≥ WCAG AA, вся навигация и аккордеон доступны с клавиатуры, `aria-label` на иконках-кнопках.
- **Браузеры:** актуальные Chrome/Firefox/Safari/Edge, мобильный Safari/Chrome (iOS/Android).
- **Аналитика:** место для сниппета Google Analytics/Яндекс.Метрика (закомментировано, не активно по умолчанию).

## 7. Контент
Копирайт готовится в двух версиях (EN — основной, RU — перевод/адаптация), тон — элегантный, заботливый. Для портфолио допустимы качественные стоковые/AI-фото, подобранные под палитру и настроение салона.

## 8. Критерии приёмки (Definition of Done)
- [ ] Все 11 секций реализованы и наполнены контентом на EN и RU
- [ ] Переключатель языка работает без перезагрузки страницы, дефолт — EN
- [ ] Адаптивность 320–1920px без горизонтального скролла
- [ ] Рабочие интерактивные элементы: бургер-меню, FAQ-аккордеон, форма/виджет записи с валидацией, sticky header, плавный скролл к якорям
- [ ] Lighthouse (мобильный): Performance/Accessibility/SEO/Best Practices ≥ 90
- [ ] Нет ошибок в консоли браузера
