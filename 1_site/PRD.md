# PRD — Лендинг «Кофейня»

## 1. Обзор и цель проекта
Портфолио-лендинг для кофейни: демонстрация умения делать быстрый, конверсионный one-page сайт для локального бизнеса. Показывается потенциальным заказчикам (владельцам кафе, фрилансерам-клиентам) как образец.

**Основная цель конверсии:** клик по CTA «Заказать онлайн» / «Посмотреть меню».
**Демонстрируемые компетенции:** современный дизайн 2026, адаптивность, двуязычность EN/RU, живые интерактивные элементы без бэкенда.

## 2. Границы проекта
**В скоуп:** одностраничный сайт (one-pager) с якорной навигацией, статичный контент, переключатель EN/RU на клиенте, форма заказа/контакта (клиентская валидация, без реальной отправки — success-состояние в UI), базовое SEO, адаптив 320–1920px.
**Вне скоупа:** реальная интеграция с системой заказов/доставки и оплатой, CMS/админка, мультистраничность, реальные пользовательские аккаунты.

## 3. Целевая аудитория
Жители района / офисные работники поблизости, любители кофе, ищущие атмосферное место или быстрый заказ на вынос. См. `PLAN.md` — полный портрет и палитра/шрифты.

## 4. Технический стек
- **HTML5 + CSS3 (без препроцессоров) + Vanilla JS (ES6+)**, без сборки — открывается напрямую в браузере, легко деплоится на любой статичный хостинг.
- Шрифты — Google Fonts (Fraunces/Playfair + Inter), подключение через `<link>`.
- Иконки — инлайн SVG (без иконочных шрифтов).
- i18n — словари `js/i18n/en.js`, `js/i18n/ru.js` + атрибуты `data-i18n`, язык по умолчанию **EN**, выбор сохраняется в `localStorage`.
- Адаптив — mobile-first, CSS Grid/Flexbox, брейкпоинты 375 / 768 / 1024 / 1440.
- Структура файлов:
  ```
  1_site/
    index.html
    css/style.css
    js/main.js
    js/i18n.js
    assets/img/
  ```

## 5. Функциональные требования по секциям
1. **Header** — лого, якорные ссылки (Меню/О нас/Галерея/Отзывы/Контакты), переключатель EN/RU, CTA «Order Online», sticky, бургер-меню на мобильном.
2. **Hero** — заголовок ≤10 слов, подзаголовок, кнопка «View Menu» (основная) + «Order to Go» (вторичная), фоновое фото/градиент-оверлей для читаемости текста.
3. **Trust bar** — 3–4 цифры (лет на рынке, чашек в день, рейтинг), простая горизонтальная строка/иконки.
4. **О нас** — блок текст+фото (2 колонки на десктопе, стек на мобильном), история и обжарка зерна.
5. **Меню** — табы или 3 карточки-категории (кофе/десерты/завтраки), в каждой — 3–4 позиции с фото, названием, ценой.
6. **Галерея** — сетка 6–8 фото (CSS grid), лайтбокс по клику необязателен (можно просто увеличение по hover).
7. **Отзывы** — 3–4 карточки (фото/аватар, имя, текст, звёзды), горизонтальный скролл на мобильном.
8. **Промо-баннер** — акция/подписка на кофе, один CTA.
9. **FAQ** — аккордеон, 4–6 вопросов, открытие одного пункта закрывает предыдущий.
10. **Контакты/CTA** — встроенная карта (iframe Google Maps placeholder), адрес, часы работы, телефон, форма (имя, телефон/email, кнопка «Отправить» с клиентской валидацией и success-сообщением).
11. **Footer** — соцсети (иконки-ссылки), копирайт, повтор навигации.

## 6. Нефункциональные требования
- **Производительность:** изображения в WebP, `loading="lazy"` для контента ниже первого экрана, минимум внешних зависимостей.
- **SEO:** уникальные `<title>`/`<meta description>`/Open Graph теги для EN и RU версий контента, семантическая разметка (`header`, `main`, `section`, `footer`), alt-тексты на всех изображениях.
- **Доступность (a11y):** контраст текста ≥ WCAG AA, вся навигация и аккордеон доступны с клавиатуры, `aria-label` на иконках-кнопках (бургер, переключатель языка).
- **Браузеры:** актуальные Chrome/Firefox/Safari/Edge, мобильный Safari/Chrome (iOS/Android).
- **Аналитика:** место для сниппета Google Analytics/Яндекс.Метрика (закомментировано, не активно по умолчанию).

## 7. Контент
Копирайт готовится в двух версиях (EN — основной, RU — перевод/адаптация), тон — тёплый, «ремесленный». Для портфолио допустимы качественные стоковые/AI-фото, подобранные под палитру и настроение кофейни.

## 8. Критерии приёмки (Definition of Done)
- [ ] Все 11 секций реализованы и наполнены контентом на EN и RU
- [ ] Переключатель языка работает без перезагрузки страницы, дефолт — EN
- [ ] Адаптивность 320–1920px без горизонтального скролла
- [ ] Рабочие интерактивные элементы: бургер-меню, FAQ-аккордеон, форма с валидацией, sticky header, плавный скролл к якорям
- [ ] Lighthouse (мобильный): Performance/Accessibility/SEO/Best Practices ≥ 90
- [ ] Нет ошибок в консоли браузера
