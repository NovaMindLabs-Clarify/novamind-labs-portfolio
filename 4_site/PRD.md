# PRD — Лендинг «Магазин цветов»

## 1. Обзор и цель проекта
Портфолио-лендинг для цветочного магазина: демонстрация умения делать быстрый, конверсионный one-page сайт для локального бизнеса. Показывается потенциальным заказчикам как образец.

**Основная цель конверсии:** клик по CTA «Выбрать букет» / «Заказать доставку».
**Демонстрируемые компетенции:** современный дизайн 2026, адаптивность, двуязычность EN/RU, живые интерактивные элементы без бэкенда.

## 2. Границы проекта
**В скоуп:** одностраничный сайт (one-pager) с якорной навигацией, статичный контент, переключатель EN/RU на клиенте, каталог букетов с ценами и кнопкой «В корзину» (UI-имитация без реальной оплаты — success-состояние), форма заказа (клиентская валидация), базовое SEO, адаптив 320–1920px.
**Вне скоупа:** реальная интеграция корзины/оплаты и логистики доставки, CMS/админка, мультистраничность, реальные пользовательские аккаунты.

## 3. Целевая аудитория
Люди, покупающие цветы к поводу (день рождения, годовщина, извинение, похороны) или оформляющие подписку на регулярные букеты; также B2B (мероприятия, свадьбы). См. `PLAN.md` — полный портрет и палитра/шрифты.

## 4. Технический стек
- **HTML5 + CSS3 (без препроцессоров) + Vanilla JS (ES6+)**, без сборки — открывается напрямую в браузере, легко деплоится на любой статичный хостинг.
- Шрифты — Google Fonts (Playfair Display + рукописный акцент + Inter), подключение через `<link>`.
- Иконки — инлайн SVG (без иконочных шрифтов).
- i18n — словари `js/i18n/en.js`, `js/i18n/ru.js` + атрибуты `data-i18n`, язык по умолчанию **EN**, выбор сохраняется в `localStorage`.
- Адаптив — mobile-first, CSS Grid/Flexbox, брейкпоинты 375 / 768 / 1024 / 1440.
- Структура файлов:
  ```
  4_site/
    index.html
    css/style.css
    js/main.js
    js/i18n.js
    assets/img/
  ```

## 5. Функциональные требования по секциям
1. **Header** — лого, якорные ссылки (Каталог/Поводы/О нас/Отзывы/Контакты), переключатель EN/RU, CTA «Order Bouquet», sticky, бургер-меню на мобильном.
2. **Hero** — эмоциональный заголовок ≤10 слов, подзаголовок, кнопка «Choose a Bouquet» (основная) + «Same-Day Delivery» (вторичная), крупное фото букета на светлом фоне.
3. **Trust bar** — 3–4 цифры (лет на рынке, букетов доставлено, рейтинг, «доставка за 2 часа»), горизонтальная строка/иконки.
4. **Поводы** — 4–5 карточек-категорий (день рождения/свадьба/извинение/просто так/подписка) с переходом к каталогу (якорь/фильтр).
5. **Каталог** — сетка 6–9 карточек букетов (фото, название, цена, кнопка «Add to Cart» с UI-имитацией добавления).
6. **О нас** — блок текст+фото (2 колонки на десктопе, стек на мобильном), философия свежести и качества.
7. **Галерея** — процесс сборки, атмосфера мастерской, сетка 6–8 фото.
8. **Отзывы** — 3–4 карточки (фото/аватар, имя, текст, звёзды), горизонтальный скролл на мобильном.
9. **FAQ** — аккордеон, 4–6 вопросов (сроки доставки, зона доставки, оплата, гарантия свежести).
10. **Доставка/контакты** — карта зоны доставки (iframe Google Maps placeholder), адрес, часы работы, телефон, форма заказа (имя, адрес, телефон, комментарий, кнопка с валидацией и success-сообщением).
11. **Footer** — соцсети (Instagram и др.), копирайт, повтор навигации.

## 6. Нефункциональные требования
- **Производительность:** изображения в WebP, `loading="lazy"` для контента ниже первого экрана, минимум внешних зависимостей.
- **SEO:** уникальные `<title>`/`<meta description>`/Open Graph теги для EN и RU версий контента, семантическая разметка, alt-тексты на всех изображениях.
- **Доступность (a11y):** контраст текста ≥ WCAG AA, вся навигация и аккордеон доступны с клавиатуры, `aria-label` на иконках-кнопках.
- **Браузеры:** актуальные Chrome/Firefox/Safari/Edge, мобильный Safari/Chrome (iOS/Android).
- **Аналитика:** место для сниппета Google Analytics/Яндекс.Метрика (закомментировано, не активно по умолчанию).

## 7. Контент
Копирайт готовится в двух версиях (EN — основной, RU — перевод/адаптация), тон — эмоциональный, тёплый, ориентированный на повод/чувства. Для портфолио допустимы качественные стоковые/AI-фото, подобранные под палитру и настроение цветочного магазина.

## 8. Критерии приёмки (Definition of Done)
- [ ] Все 11 секций реализованы и наполнены контентом на EN и RU
- [ ] Переключатель языка работает без перезагрузки страницы, дефолт — EN
- [ ] Адаптивность 320–1920px без горизонтального скролла

- [ ] Lighthouse (мобильный): Performance/Accessibility/SEO/Best Practices ≥ 90
- [ ] Нет ошибок в консоли браузера
